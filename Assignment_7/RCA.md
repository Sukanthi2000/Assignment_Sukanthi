| **Incident Summary**    | Flask app fails randomly, sometimes returning 500 errors or crashing on `/data` endpoint.                                                                                                                                    |
|**Steps to Reproduce**   | 1. Build and run Docker container (`docker run -p 5000:5000 buggy-app`) <br> 2. Access `http://localhost:5000/data` multiple times <br> 3. Observe random failures                                                           |
| **Observed Behavior**   | - Random 500 responses <br> - AttributeError in `/data` endpoint: `'NoneType' object has no attribute 'length'` <br> - Console logs: `Simulated random failure!`                                                             |
| **Root Cause**          | - Middleware randomly returns 500 (simulating instability) <br> - `/data` endpoint tries to access `.length` of `None` variable causing crash                                                                                |
| **Evidence (Logs)**     |`Simulated random failure!` <br> `Traceback (most recent call last): AttributeError: 'NoneType' object has no attribute 'length'`                             |
| **Proposed Fix**        | - Remove random failure middleware for production <br> - Add input validation in `/data` to avoid `None` errors <br> - Add proper error handling with `try/except` <br> - Optionally, implement logging and retry mechanisms |
| **Impact & Mitigation** | - Random failures could affect users and downstream services <br> - Mitigation: temporary patch with error handling, alerting for failures, and test fixes in dev environment                                                |
